# User Story Template

**Title:** Administrator role.
As a Administrator, I want to control access and functionality to the Capstone system for grant permissions to Doctor and Patients rolls, so that Doctors can see reserved appointments by patients, and patients can make reservations with specialist doctors.

**Acceptance Criteria:**
1. Administrator can make same activities like Doctors and Patients rolls.
2. Administrator is the suer user in the Capstone system.
3. Administrator can modify the records made by Doctors and Patients rolls.

**Priority:** Low.
**Story Points:** 5
**Notes:**
- Administrator is the main user in the Capstone system.

**Title:** Doctor role.

As a Doctor, I want to handle my own availability for Patients, recording dates and times for attending medical services, and listing reserved appointments by patients, so that I can give medical services successfully.

**Acceptance Criteria:**

1. Doctors can make changes to reserved appointments made by patients.
2. Doctors can list reserved appointments.
3. Doctors can cancel reserved appointments.
   
**Priority:** Low.
**Story Points:** 5
**Notes:**
- Doctors is an operator in the Capstone system.

**Title:** Patient role.

As a Patient, I want record, list and modify reserved appointments with medical doctors, so that I can receive medical attention for specific illness.

**Acceptance Criteria:**

1. Patients can make appointment reservations for medical attention.
2. Patients can list appointment reservations in the Capstone system. 
3. Patients can cancel medical appointment in the Capstone system.
   
**Priority:** Low.
**Story Points:** 5
**Notes:**
- Patient is an operator in the Capstone system.
