# User Story Template

**Title:** Administrator role.

History user 1:

As a Administrator, I want to control access and functionality to the Capstone system for grant permissions to Doctor and Patients rolls, so that Doctors can see reserved appointments by patients, and patients can make reservations with specialist doctors.

**Acceptance Criteria:**
1. Administrator can make same activities like Doctors and Patients rolls.
2. Administrator is the suer user in the Capstone system.
3. Administrator can modify the records made by Doctors and Patients rolls.

**Priority:** Low.
**Story Points:** 5
**Notes:**
- Administrator is the main user in the Capstone system.
